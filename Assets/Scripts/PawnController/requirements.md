# PawnController Requirements

## Общая концепция

Система управления персонажами через NavMesh с интерфейсом для обработки пользовательского ввода и автоматических тестов.

## Архитектура

### Структура компонентов

```
PawnController (MonoBehaviour на общем контроллере)
    ↓ использует
ISelector (интерфейс)
    ↓ реализуют
├── AutoSelector (для автоматических тестов)
└── InputActionsMapper (для пользовательского ввода)
    ↓ генерируют события →
Event Handler (обработчик событий)
```

### Основной принцип

1. **PawnController** - главный компонент, висит на общем контроллере, использует интерфейс ISelector
2. **ISelector** - интерфейс для генерации событий выбора
3. **Реализации ISelector**:
   - **AutoSelector** - автоматическая генерация событий для тестов
   - **InputActionsMapper** - преобразование пользовательского ввода в события
4. **Event Handler** - класс, который применяет/обрабатывает события и содержит логику управления персонажем

### Разделение ответственности

- **ISelector и его реализации** - отвечают за генерацию событий (что произошло)
- **Event Handler** - отвечает за обработку событий (что делать)
- **PawnController** - координирует работу, связывает ISelector с Event Handler

## Интерфейс событий

### Интерфейс ISelector

Интерфейс должен предоставлять события для подписки. Реализации интерфейса (AutoSelector, InputActionsMapper) вызывают эти события.

### События:

1. **onSelect(PawnBrainNavMesh pawn)**

   - Вызывается при выборе персонажа/агента
   - Передает ссылку на выбранного персонажа
   - Генерируется при клике по персонажу или автоматически в тестах

2. **onNavClick(Vector3 position)**

   - Вызывается при клике по NavMesh или пустому пространству
   - Передает мировые координаты точки клика (или ближайшей точки на NavMesh)
   - Генерируется при клике мыши по NavMesh или автоматически в тестах

3. **onCancel()**
   - Вызывается при нажатии ESC или отмене действия
   - Используется для сброса выбора персонажа
   - Генерируется при нажатии ESC или автоматически в тестах

### Реализации ISelector

**InputActionsMapper:**

- Слушает пользовательский ввод (клики мыши, ESC)
- Преобразует ввод в события onSelect, onNavClick, onCancel
- Работает с камерой для raycast'ов
- Работает с NavMesh для поиска точек

**AutoSelector:**

- Генерирует события автоматически по заданной последовательности
- Использует данные из инспектора (список точек, персонажи, задержки)
- Используется для автоматических тестов

### Приоритет событий

- Клик по персонажу имеет приоритет над кликом по NavMesh
- Если персонаж уже выбран и происходит клик по другому персонажу - переключить выбор (если управление кликов не перехвачено)

## Функциональность

### 1. Выбор персонажа

- Клик по персонажу выбирает его
- Визуальная индикация выбранного персонажа (на усмотрение реализации)
- При выборе нового персонажа предыдущий выбор отменяется (если такое действие разрешено)

### 2. Предпросмотр пути

- При выбранном персонаже и движении мыши показывать путь к текущей позиции мыши
- Путь рассчитывается через NavMesh (NavMeshAgent.CalculatePath или NavMesh.CalculatePath)
- Визуализация пути (линия, стрелки, и т.д. - на усмотрение реализации)

### 3. Перемещение персонажа

- Клик по NavMesh при выбранном персонаже отправляет его в эту точку
- Использовать NavMeshAgent.SetDestination() для перемещения
- После перемещения персонаж **не** остается выбранным

### 4. Отмена выбора

- ESC или OnCancel() снимает выбор с персонажа
- Скрывает предпросмотр пути

## Компоненты системы

### PawnController

- MonoBehaviour компонент, висит на общем контроллере сцены
- Содержит ссылку на ISelector (назначается в инспекторе или через код)
- Содержит ссылку на Event Handler (обработчик событий)
- Подписывается на события из ISelector
- Передает события в Event Handler для обработки

### Event Handler (обработчик событий)

- Класс, который содержит логику управления персонажем
- Обрабатывает события: onSelect, onNavClick, onCancel
- Управляет выбранным персонажем
- Управляет предпросмотром пути
- Отправляет команды перемещения персонажам

## Автоматические тесты

### Реализация через AutoSelector

- AutoSelector реализует ISelector
- В инспекторе задается последовательность действий для тестов
- При запуске (если включено) AutoSelector генерирует события автоматически
- PawnController переключается между InputActionsMapper и AutoSelector

### Структура данных для тестов

- Список последовательностей действий:
  - Выбор персонажа (onSelect)
  - Точки перемещения (onNavClick)
  - Отмена (onCancel)
- Задержки между действиями (опционально)
- Флаг включения автотестов

## Технические детали

### Поиск точки на NavMesh

- Использовать NavMesh.SamplePosition() для нахождения ближайшей точки на NavMesh от клика
- Raycast от камеры в мировое пространство
- Если точка не на NavMesh - найти ближайшую доступную точку в заданном радиусе

### Расчет пути

- Использовать NavMesh.CalculatePath() или NavMeshAgent.CalculatePath()
- Извлекать угловые точки через path.corners
- Проверять валидность пути перед отображением

### Визуализация пути

- Обновлять предпросмотр пути при движении мыши
- Показывать валидность пути цветом
- Скрывать путь при отмене выбора или клике

## Зависимости

- Unity AI Navigation (NavMesh система)
- PawnBrainNavMesh (класс персонажа)
- Input System (для обработки кликов и ESC)

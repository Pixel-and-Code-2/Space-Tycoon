# Space Tycoon

Супер новая и современная игра про космическую станцию (_дополненное описание_, кнопочку "развернуть" самостоятельно находим). Подробнее [ниже](#spacetycoon)

## Структура проекта

```
Assets/                                     # место хранения всех частей игры
    Prefabs/                                # сохранённые сложные переиспользуемые объекты
    Scenes/                                 # сцены (зачастую разные для разных тестов)

    Scripts/                                # папка со всеми C# скриптами проекта
        CameraController/                   # группа скриптов для работы камеры
        ModulesScripts (unattachable)/      # папка со вспомогательными скриптами для модулей

    Settings/                               # настройки проекта: input bindings, graphics
.gitignore
README.md                                   # вы находитесь здесь)
```

## Требования к документации

### 1. Базовый синтаксис

XML-комментарии размещаются непосредственно перед элементом, который они документируют (класс, метод, свойство и т.д.).

```csharp
/// <summary>
/// Это краткое описание того, что делает этот метод.
/// </summary>
/// <param name="input">Описание параметра 'input'.</param>
/// <returns>Описание возвращаемого значения.</returns>
public int CalculateSum(int input)
{
    return input + 5;
}
```

### 2. Основные теги документации

| Тег              | Назначение                                                                  | Пример использования                                                                                        |
| :--------------- | :-------------------------------------------------------------------------- | :---------------------------------------------------------------------------------------------------------- |
| `<summary>`      | **Обязательный.** Краткое описание элемента.                                | `<summary>Вычисляет факториал заданного числа.</summary>`                                                   |
| `<param>`        | Описание параметра метода или конструктора. Требует атрибута `name`.        | `<param name="number">Исходное положительное число.</param>`                                                |
| `<returns>`      | Описание возвращаемого значения метода.                                     | `<returns>Факториал числа в виде целого числа.</returns>`                                                   |
| `<exception>`    | Описание исключения, которое может быть выброшено. Требует атрибута `cref`. | `<exception cref="System.ArgumentOutOfRangeException">Выбрасывается, если число отрицательное.</exception>` |
| `<remarks>`      | Более подробное описание, которое дополняет `<summary>`.                    | `<remarks>Этот метод использует итеративный подход.</remarks>`                                              |
| `<typeparam>`    | Описание обобщенного (generic) типа. Требует атрибута `name`.               | `<typeparam name="T">Тип элемента в коллекции.</typeparam>`                                                 |
| `<typeparamref>` | Ссылка на имя обобщенного типа внутри текста (например, в `<remarks>`).     | `... для типа <typeparamref name="T"/> ...`                                                                 |
| `<see>`          | Встраиваемая ссылка на другой элемент (например, метод или класс).          | `См. <see cref="AnotherClass.DoSomething()"/>.`                                                             |
| `<seealso>`      | Элемент, который будет отображаться в разделе "См. также".                  | `<seealso cref="System.String"/>`                                                                           |

### 3. Использование ссылок (`cref`)

Атрибут `cref` (Canonical Reference) используется для создания ссылок на другие типы, члены или методы в вашем коде или в стандартной библиотеке .NET. Компилятор проверяет эти ссылки на корректность.

**Примеры `cref`:**

- **Метод:** `cref="MyClass.MyMethod(System.Int32)"`
- **Тип:** `cref="System.String"`
- **Свойство:** `cref="MyClass.Count"`

**Важно:** Если вы используете `cref`, компилятор может выдать предупреждение, если ссылка не найдена, что помогает поддерживать актуальность документации.

### 4. Документирование различных элементов

#### А. Классы и структуры

```csharp
/// <summary>
/// Представляет собой базовый класс для всех геометрических фигур.
/// </summary>
/// <remarks>
/// Этот класс не должен использоваться напрямую; используйте его производные классы.
/// </remarks>
public abstract class Shape
{
    // ...
}
```

#### Б. Методы

```csharp
/// <summary>
/// Вычисляет площадь фигуры.
/// </summary>
/// <param name="unit">Единица измерения (например, "cm" или "m").</param>
/// <returns>Площадь фигуры в виде числа с плавающей точкой.</returns>
/// <exception cref="ArgumentException">Выбрасывается, если переданная единица измерения не поддерживается.</exception>
public abstract double CalculateArea(string unit);
```

#### В. Свойства

```csharp
/// <summary>
/// Получает или устанавливает уникальный идентификатор объекта.
/// </summary>
/// <value>Уникальный GUID.</value>
public Guid Id { get; set; }
```

#### Г. Перечисления (Enums)

Для перечислений тег `<summary>` применяется к самому перечислению, а теги `<param>` заменяются тегами `<member name="F:Namespace.EnumName.ValueName">` (хотя часто достаточно простого комментария над каждым значением, или использования `<summary>` для самого перечисления).

```csharp
/// <summary>
/// Определяет уровни доступа пользователя.
/// </summary>
public enum AccessLevel
{
    /// <summary>
    /// Полный доступ ко всем функциям.
    /// </summary>
    Administrator,

    /// <summary>
    /// Ограниченный доступ только для чтения.
    /// </summary>
    Reader
}
```

### 5. Использование тега `<example>`

Тег `<example>` полезен для демонстрации того, как использовать документируемый элемент. Вы можете использовать вложенные теги `<code />` для форматирования кода.

```csharp
/// <summary>
/// Прибавляет 10 к заданному числу.
/// </summary>
/// <param name="value">Исходное число.</param>
/// <returns>Число плюс 10.</returns>
/// <example>
/// Следующий пример демонстрирует использование метода AddTen:
/// <code>
/// var calculator = new MyMathClass();
/// int result = calculator.AddTen(5); // result будет равен 15
/// Console.WriteLine(result);
/// </code>
/// </example>
public int AddTen(int value)
{
    return value + 10;
}
```
